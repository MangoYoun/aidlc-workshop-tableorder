# User Personas

## Persona 1: 고객 (Customer)

### 기본 정보
- **이름**: 김민수 (대표 고객)
- **역할**: 매장 방문 고객
- **연령대**: 20-50대
- **기술 수준**: 기본적인 스마트폰/태블릿 사용 가능

### 주요 목표
1. **빠르고 편리한 주문**: 직원을 기다리지 않고 즉시 주문하고 싶음
2. **메뉴 정보 확인**: 메뉴 사진과 설명을 보고 선택하고 싶음
3. **주문 내역 확인**: 내가 주문한 메뉴와 금액을 확인하고 싶음
4. **추가 주문 용이성**: 식사 중 추가 주문을 쉽게 하고 싶음

### 핵심 니즈
- 직관적이고 사용하기 쉬운 인터페이스
- 메뉴 사진과 상세 정보
- 주문 확인 및 수정 기능
- 빠른 응답 속도

### 페인 포인트
- 직원을 불러서 주문하는 번거로움
- 메뉴판만으로는 메뉴 정보가 부족함
- 주문 후 추가 주문이 어려움
- 주문 내역을 확인하기 어려움

### 사용 패턴
- 테이블에 앉아서 태블릿으로 메뉴 조회
- 여러 메뉴를 장바구니에 담고 한 번에 주문
- 식사 중 추가 주문
- 주문 내역 확인

## Persona 2: 관리자 (Admin)

### 기본 정보
- **이름**: 박영희 (매장 운영자)
- **역할**: 매장 관리자 / 운영자
- **연령대**: 30-60대
- **기술 수준**: 기본적인 컴퓨터/태블릿 사용 가능

### 주요 목표
1. **실시간 주문 관리**: 들어오는 주문을 즉시 확인하고 처리하고 싶음
2. **효율적인 운영**: 테이블별 주문 상태를 한눈에 파악하고 싶음
3. **메뉴 관리**: 메뉴 정보를 쉽게 추가/수정/삭제하고 싶음
4. **테이블 관리**: 테이블 세션을 관리하고 과거 내역을 조회하고 싶음

### 핵심 니즈
- 실시간 주문 알림 및 모니터링
- 테이블별 주문 현황 대시보드
- 간편한 메뉴 관리 기능
- 주문 상태 변경 및 삭제 권한
- 테이블 세션 관리 기능

### 페인 포인트
- 주문이 밀릴 때 관리가 어려움
- 테이블별 주문 내역 파악이 어려움
- 메뉴 변경 시 번거로움
- 고객이 떠난 후 테이블 정리가 수동적임

### 사용 패턴
- 관리자 대시보드에서 실시간 주문 모니터링
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 테이블 이용 완료 처리
- 메뉴 추가/수정/삭제
- 과거 주문 내역 조회
